# Objectives

## General Objective

Design and validate a Proof of Concept (PoC) of a multiagent system capable of analyzing software repositories to identify technical debt associated with a specific Non-Functional Requirement (NFR), focusing on maintainability, and generating AI-based recommendations.

---

## Specific Objectives

1. Define the conceptual relationship between technical debt and non-functional requirements, with emphasis on maintainability.

2. Identify measurable indicators of technical debt related to maintainability, such as:
   - Cyclomatic complexity
   - Code duplication
   - Class and method size
   - Coupling and cohesion

3. Design a multiagent architecture for:
   - Repository ingestion
   - Static analysis
   - Technical debt detection
   - Recommendation generation
   - Report generation

4. Integrate AI-based components (LLMs) to enhance:
   - Interpretation of code quality issues
   - Generation of contextualized recommendations

5. Evaluate the effectiveness of the proposed approach using selected repositories and defined metrics.

6. Analyze the feasibility of extending the approach to other non-functional requirements in future work.
