# Experiment Design

## Objective

Evaluate the effectiveness of a multiagent system in detecting technical debt associated with the Maintainability NFR.

---

## Approach

The experiment compares:

1. Traditional static analysis tools
2. Proposed multiagent AI-based system

---

## Input Data

- Public GitHub repositories
- Different sizes and complexity levels
- Multiple programming languages (optional)

---

## Experimental Steps

1. Select repository
2. Run static analysis tools
3. Run multiagent system
4. Collect outputs
5. Compare results

---

## Evaluation Metrics

- Number of detected issues
- Relevance of detected issues
- Quality of recommendations
- Explainability
- Consistency

---

## Expected Outcome

The multiagent system should:

- Detect more context-aware technical debt
- Provide actionable recommendations
- Improve interpretability compared to traditional tools
